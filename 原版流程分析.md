# SnowDesktop 原版全流程分析

## 1. 启动流程 (Run方法, L340~509)

```
1. SetProcessDpiAwarenessContext(PER_MONITOR_AWARE_V2)
2. InitCommonControlsEx
3. OleInitialize
4. InitializeGraphics          ← D3D11 + D2D + DComp + DWrite 全部初始化
5. InitializeShell             ← SHGetDesktopFolder + SHGetSpecialFolderLocation(CSIDL_DESKTOP)
6. LoadLayoutSlots             ← 从 JSON 加载布局记录
7. FindDesktopWindows          ← 找 Progman→DefView→SysListView32
8. 隐藏 SysListView32          ← SetProp+ShowWindow(SW_HIDE)
9. RegisterWindowClass         ← WNDCLASSEXW, hbrBackground=nullptr, CS_DBLCLKS
10. RegisterWindowMessage("TaskbarCreated")
11. CreateControlWindow        ← 接收托盘消息的隐藏窗口
12. CreateDesktopWindow        ← WS_CHILD 挂在 desktop host 上, 创建 DComp target+surface
13. RegisterOleDropTarget
14. AddTrayIcon
15. ReloadItems                ← IShellFolder::EnumObjects 枚举图标, 加载 layout, LayoutItems
16. RegisterShellChangeNotifications
17. 启动定时器(回收站轮询/桌面host监控/Widget刷新)
18. 初始化 SettingsWindow, WidgetEngine(Lua)
19. 消息循环
```

**关键**: InitGraphics 在窗口创建之前; CreateCompositionTarget 在 CreateDesktopWindow 末尾(窗口显示后)。

## 2. 窗口系统

### 窗口层次
```
Progman (桌面宿主)
├── SHELLDLL_DefView
│   └── SysListView32          ← Explorer 原生图标层(被隐藏)
└── SnowDesktopNativeProofWindow ← WS_CHILD, 我们自己创建的覆盖层
```

还有一个隐藏的 controlHwnd_(控制窗口)用于接收托盘消息。

### 窗口类注册 (RegisterWindowClass)
- WNDCLASSEXW
- cbSize = sizeof(WNDCLASSEXW)
- style = CS_DBLCLKS
- hbrBackground = nullptr       ← 无背景画刷! DComp 直接渲染
- hCursor = LoadCursor(IDC_ARROW)
- lpszClassName = "SnowDesktopNativeProofWindow"

### CreateDesktopWindow
```
1. virtualLeft/Top/Width/Height = GetSystemMetrics(SM_X/Y/CX/CYVIRTUALSCREEN)
2. UpdateLayoutWorkArea()
3. parent = desktopWindows_.host ?? GetDesktopWindow()
4. origin = ScreenToClient(parent, {virtualLeft, virtualTop})
5. CreateWindowExW(WS_EX_TOOLWINDOW, WS_CHILD|WS_VISIBLE, origin, size, parent)
6. 如果失败 → CreateWindowExW(WS_POPUP) + SetParent + 改 style 为 WS_CHILD
7. SetWindowPos(HWND_TOP or HWND_BOTTOM)
8. ShowWindow + UpdateWindow
9. LoadAppIcon → WM_SETICON
10. CreateCompositionTarget → CreateTargetForHwnd(FALSE) + CreateVisual + CreateOrResizeCompositionSurface
```

**关键**: CreateTargetForHwnd 第二参数是 FALSE(子窗口 visual)。

## 3. 图形初始化 (InitializeGraphics, 不含创建Target/Surface)

```
1. D3D11CreateDevice(HARDWARE → WARP 回退)
2. D2D1CreateFactory(SINGLE_THREADED)
3. d3dDevice→GetDXGIDevice → CreateDevice(CreateDeviceContext)
4. bitmapContext_->SetDpi(96,96) + SetUnitMode(PIXELS)
5. DCompositionCreateDevice2(d2dDevice.Get())  ← 传入 D2D 设备,共享 D3D11 设备
6. DWriteCreateFactory → CreateTextFormat × 4种
7. CreateStrokeStyle(dotted)
```

**注意**: InitGraphics 不管 CreateTargetForHwnd / CreateVisual / CreateSurface, 那些在 CreateCompositionTarget 里。

## 4. DComp 渲染流程

### 每帧渲染路径
```
WM_PAINT → BeginPaint/EndPaint → DrawD2D()

DrawD2D():
  1. CreateOrResizeCompositionSurface()   ← 按需创建/重建 surface
  2. dcompSurface_->BeginDraw(&rawContext, &updateOffset)
  3. context.Attach(rawContext)
  4. context->SetDpi(96,96)
  5. context->SetUnitMode(PIXELS)
  6. context->SetTransform(Translation(updateOffset))
  7. context->Clear(ColorF(0,0,0,0))       ← 透明清理
  8. DrawD2DScene(context)                  ← 实际绘制
  9. context->SetTransform(Identity)
  10. context.Reset()
  11. dcompSurface_->EndDraw()
  12. dcompDevice_->Commit()
```

### CreateOrResizeCompositionSurface
```
1. GetClientRect(hwnd_)
2. 如果 surface 存在且尺寸匹配 → 返回 S_OK(不重建)
3. dcompDevice_->CreateSurface(width, height, B8G8R8A8_UNORM, PREMULTIPLIED)
4. dcompVisual_->SetContent(surface)
5. 缓存 width/height 到 compositionWidth_/compositionHeight_
```

**关键**: PREMULTIPLIED alpha, 尺寸变化时完全重建 surface。

### DrawD2DScene (L9074)
```
1. 如果 reloading_ → return
2. 遍历 items_: 
   - 跳过不可见的(item.bounds为空或不在可见区)
   - DrawD2DItemAt(context, item, item.bounds, item.selected)
3. DrawD2DDraggedItems(context)    ← 拖拽中的偏移绘制
4. DrawD2DWidgets(context)         ← 组件渲染
5. DrawD2DStatus(context, client)  ← 状态栏
6. 如果拖拽中: DrawD2DDropTarget(context)  ← 插入线/placeholder
7. DrawD2DNavigationPanel(context) ← 页面导航
8. 如果框选中: DrawD2DMarquee(context)
```

### DrawD2DItemGeneric (单个图标渲染, L10673)
```
1. hover? → 半透明圆角矩形
2. selected? → 灰色选中框(GetItemSelectionRect)
3. GetOrCreateD2DBitmap(hbm) → DrawBitmap(iconRect)
4. CreateTextLayout(name) → DrawTextLayout(带阴影)
```

## 5. 图标布局系统

### 关键数据结构
- `items_` (vector<DesktopItem>) — 所有桌面项
- `gridPages_` (vector<GridPage>) — 每屏一个 GridPage
- `layoutRecords_` (map<key, LayoutRecord>) — 持久化布局

### GridPage 字段
- id, monitorId
- bounds / workArea (相对虚拟屏坐标, 不是屏幕坐标!)
- columns, rows
- cellWidth(kCellWidth=92), cellHeight(kMinCellHeight=136)
- gapX, gapY
- marginX(kGridMarginX=6), marginY(kGridMarginY=6)

### 布局计算链
```
UpdateLayoutWorkArea():
  1. 用 virtualLeft/Top/Width/Height 构建 layoutWorkArea_
  2. EnumDisplayMonitors → EnumGridPageMonitorProc
     - 每个显示器一个 GridPage, bounds/workArea 相对于 virtualLeft/Top
  3. 如果没用 → 一个 fallback 页面
  4. 按 bounds.left 排序页面
  5. 对每个页面: clamp workArea + ConfigureGridPage
  
ConfigureGridPage(page):
  - columns = usableWidth / cellW
  - rows = usableHeight / cellH  
  - gapX = (剩余空间) / (columns-1)

LayoutItems():
  - 对每个 item: 
    - 确保 pageId 有效
    - 根据 gridCell + gridSpan 计算 bounds
    - GetGridRect(gridCell, gridSpan) → RECT

GetGridRect(gridCell, gridSpan):
  - 找到对应 GridPage
  - 从 margin+cell+gaps 计算绝对像素坐标
```

### 图标矩形计算
```
GetItemIconRect(bounds):
  - cellH < 50 (列表模式): 左侧 32px 图标
  - 标准模式: iconSize = min(cellW-8, cellH-kTextHeight-8)
  - iconX = bounds.left + (cellW - iconSize) / 2
  - iconY = bounds.top + 2

GetItemTextRect(bounds, expanded):
  - textTop = iconRect.bottom + 2
  - 展开: kTextExpandedHeight(58), 折叠: kTextCollapsedHeight(34)

GetItemSelectionRect(bounds, expanded):
  - Union(iconRect, textRect) + 4px padding
```

## 6. 桌面项加载 (ReloadItems → L2005~2200)

```
1. 如果 reloading_ → return
2. reloading_ = true
3. 如果 reloadLayoutFromDisk → LoadLayoutSlots()
4. items_.clear() + d2dIconCache_.clear()
5. 获取 userDesktopPath/commonDesktopPath/userProfilePath
6. desktopFolder_->EnumObjects(SHCONTF_FOLDERS|SHCONTF_NONFOLDERS|SHCONTF_INCLUDEHIDDEN)
7. 对每个 PIDL:
   a. ILCombine(desktopPidl, child) → absolute PIDL
   b. 跳过 SFGAO_HIDDEN|SFGAO_NONENUMERATED (非桌面图标)
   c. SHGetFileInfoW(PIDL, SHGFI_PIDL|SHGFI_SYSICONINDEX|SHGFI_DISPLAYNAME|SHGFI_TYPENAME)
   d. ResolveDesktopIconClsid → 判断是否特殊图标(此电脑/回收站/控制面板/网络/用户文件夹等)
   e. IsVisibleByDesktopIconSettings → 检查注册表可见性
   f. 非特殊图标: 检查文件路径是否在 userDesktopPath 或 commonDesktopPath 下
   g. GetHighResolutionShellIconBitmap(PIDL, iconIndex) → D2D bitmap
   h. 检测 .lnk → shortcutArrow
   i. GetStableLayoutKey → layoutRecords_ 查找或分配 gridCell
   j. 如果没有布局记录 → 自动分配到最近空 cell
8. 排序 → LayoutItems() → 填充 item.bounds
9. 加载 Widgets 数据
10. reloading_ = false
```

**过滤规则**:
- 特殊桌面图标(此电脑/回收站/控制面板/网络/个人文件夹): 注册表 CLSID 可见性决定
- 用户文件夹(音乐/下载/图片/视频/文档): ResolveDesktopIconClsid 返回对应 CLSID, 需要检查 IsVisibleByDesktopIconSettings
- 普通文件/快捷方式: 必须物理位于 Desktop 文件夹下(userDesktopPath 或 commonDesktopPath)

## 7. 窗口消息处理 (WindowProcThunk → HandleMessage)

关键消息:
- **WM_PAINT**: BeginPaint/EndPaint → DrawD2D
- **WM_ERASEBKGND**: return 1 (不擦除)
- **WM_SIZE**: 更新 virtualWidth/Height; 如果 attachedToDesktopHost → SetWindowPos 调整
- **WM_LBUTTONDOWN/UP/MOUSEMOVE**: 拖拽逻辑
- **WM_KEYDOWN**: Escape 退出/取消拖拽; F2/F5 重命名/刷新
- **WM_CONTEXTMENU**: 右键菜单
- **WM_TIMER**: 回收站轮询/桌面host监控/Widget刷新
- **kShellChangeMessage(WM_APP+2)**: Shell 变更通知 → 防抖后 ReloadItems
- **WM_DISPLAYCHANGE/WM_DEVICECHANGE**: 显示器变更 → UpdateLayoutWorkArea → LayoutItems → InvalidateRect

## 8. 拖拽系统 (当前过程式实现)

### 状态变量
- `draggingItems_` — 是否正在拖拽
- `selfDragActive_` — 内部拖拽(自己的 OLE DragDrop)
- `dragTargetCell_` — 目标 grid cell
- `dragHint_` — 提示文字
- `dragCurrentPoint_` / `mouseDownPoint_` — 拖拽位置

### 流程
```
OnMouseDown → 检测点击目标(DesktopHitKind)
  - Item 上 → 开始拖拽
  - Widget 标题栏 → 移动 Widget
  - Widget 边缘 → 调整大小

OnMouseMove(拖拽中) →
  - HitTestDesktop → DesktopHitKind
  - MakeInternalDragHint → 提示文字
  - 更新 dragTargetCell_ / dragHint_
  - InvalidateRect → 重绘

OnLeftButtonUp(拖拽中) →
  - 执行拖放操作(MoveToCell/CopyFiles/ShellExecuteEx 等)
  - 清理拖拽状态
```

## 9. Widget 系统

### Widget 类型
- Collection(集合) — 图标集合,可拖入
- FileCategories(桌面文件) — 按类型分类
- FolderMapping(映射文件夹) — 文件夹视图
- LuaScript — Lua 自渲染组件

### 数据流
- 每个 DesktopWidget 有 itemKeys(Collection/FileCategories) 或 folderEntries(FolderMapping)
- folderEntries 从 sourceFolderPath 读取
- Widget 布局通过 DesktopWidget.gridCell/gridSpan → GetGridRect
