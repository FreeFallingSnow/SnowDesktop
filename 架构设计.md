# SnowDesktop 拖拽系统 OO 架构设计

## 一、设计目标

消除 `app.h` 中过程式 `if/else` + `switch(ctype)` 分支，改为多态分发。每个类自己负责自己的行为。

## 二、类层次总览

```
Item（可拖拽实体，负责自渲染和 OLE）
├── DesktopIcon : Item          ← 包装 DesktopItem
│
├── Widget : Item               ← 组件基类（纯 Item，可拖拽）
│   ├── LuaScript : Widget      ← Lua 自渲染，不可放
│   │
│   └── WidgetContainer : Widget, Container  ← 组件 + 可放
│       ├── Collection : WidgetContainer
│       ├── FileCategories : WidgetContainer
│       └── FolderMapping : WidgetContainer

Container（可放置 Item 的东西，负责 BuildSlots + OnItemsDropped）
├── GridContainer : Container            ← 二维网格，可放任意 Item + Widget
│   └── DesktopGrid : GridContainer      ← 桌面，唯一可放置 Widget 的容器
│
├── ListContainer : Container            ← 一维列表，可放普通 Item，不可放 Widget
│   └── WidgetContainer（通过多重继承）

Slot（Container 内的一个位置，兼做 drop target）
├── item: Item*                 ← null = 空 slot
├── HitTest(pt)                 ← 返回命中的子区域：None / Empty / SortBefore / SortAfter / Handoff
├── GetDropHint(region, source) ← "移动到此空位" / "交给xx处理" / "重新排序" / ...
├── ExecuteDrop(region, source, mods) → Container::OnItemsDropped
└── DrawDropIndicator(ctx, pt)  ← VBar（Grid） / HBar（List） / 不画（Handoff）

注意：没有独立的 DropZone 类——Slot 自己就是 drop target。
插入线的前/后由 HitRegion 区分（SortBefore / SortAfter），无需额外对象。

注意：没有独立的 DragSource / DragOperation 类。
拖拽状态（dragItems_, dragOrigin_, dragTargetSlot_, dragTargetRegion_）由 app 直接管理。
```

## 三、核心接口定义

### 3.1 Item

```cpp
class Item {
public:
    virtual ~Item() = default;
    virtual std::wstring GetTitle() const = 0;         // 显示名
    virtual std::wstring GetPath() const = 0;           // 文件系统路径
    virtual HBITMAP GetIconBitmap() const = 0;          // 图标
    virtual RECT GetBounds() const = 0;                 // 当前位置
    virtual bool IsSelected() const = 0;
    virtual Container* GetContainer() const = 0;        // 所属容器（null=桌面）

    // 渲染自己到指定矩形 + 状态
    virtual void Draw(ID2D1DeviceContext*, RECT rect, int state) = 0;
    // state: 0=normal, 1=hovered, 2=selected, 3=dragged

    // OLE
    virtual ComPtr<IDataObject> CreateDataObject() = 0;
};
```

**DesktopIcon**：持有 `DesktopItem*`（非拥有），从 `DesktopItem` 的 name/pidl/bitmap 取值。

**决策点**：DesktopItem 和 FolderEntry 不改成继承 Item。原因：(1) 它们包含 PIDL/Shell 耦合，移动/拷贝操作复杂；(2) 改布局会影响 JSON 序列化。用适配器包装：`DesktopIcon : Item` 持 `DesktopItem*`，`FolderEntryIcon : Item` 持 `FolderEntry*`。

### 3.2 Container

```cpp
class Container {
public:
    virtual ~Container() = default;
    virtual std::wstring GetTitle() const = 0;

    // 布局
    virtual std::vector<std::unique_ptr<Slot>> BuildSlots() = 0;
    BarStyle GetInsertionStyle() const;  // 由子类覆写

    // 拖入响应（双分派：目标容器决定如何处理来自源容器的 drop）
    virtual void OnItemsDropped(const std::vector<Item*>& sourceItems, Container* origin,
        Slot* targetSlot, HitRegion region, int mods) = 0;

    // 渲染
    virtual void DrawChrome(ID2D1DeviceContext*, POINT mousePt) {}
    virtual RECT GetBounds() const = 0;
};
```

**GridContainer**：二维 col×row 网格布局。`BuildSlots()` 遍历所有 GridPage 的所有 cell。插入线默认 VBar。只有 `DesktopGrid` 使用。

**GridContainer 的能力**：
- 接受任意 Item：文件图标、文件夹条目**以及整个 Widget**
- Widget 放置走 `PlaceWidgetWithDisplacement`（沿用现有逻辑）

**ListContainer**：一维列表布局。`BuildSlots()` 按 index × itemHeight 计算每行 RECT。插入线默认 HBar。所有 Widget 使用。

**ListContainer 的限制**：不接受 Widget——只能放普通 Item。Widget 之间不能嵌套。

ListContainer 三种渲染形态（运行时根据 data 状态决定）：

| 形态 | 条件 | 行为 |
|---|---|---|
| **FixedList** | 总高度 ≤ 可见区域 | 全部可见，无滚动条，插入指示直接落在 slot |
| **ScrollingIconList** | 总高度 > 可见区域，图标模式 | 有滚动条，`BuildSlots` 减去 `scrollOffset`，插入线在图标间 |
| **ScrollingRowList** | 总高度 > 可见区域，列表模式 | 有滚动条，行高固定（32px），插入线在行间 |

**可放置 Widget 的对比**：

| Container 类型 | 可放文件/条目 | 可放 Widget |
|---|---|---|
| GridContainer (DesktopGrid) | ✅ | ✅ |
| ListContainer (Widget) | ✅ | ❌ |

**DesktopGrid**：持有 `std::vector<GridPage>*` 和 `std::vector<DesktopItem>*`（引用 app 数据）。`BuildSlots()` 遍历所有页面所有 cell。`OnItemsDropped` 检测 source 是否包含 Widget，若是则走 `PlaceWidgetWithDisplacement`。

**WidgetContainer 子类**：
- `Collection::OnItemsDropped` → 调用 `app->MoveSelectedItemsToCollection`
- `FileCategories::OnItemsDropped` → 调用 `app->AddSelectedItemsToFileCategoryWidget`
- `FolderMapping::OnItemsDropped` → 同源重排 `FinishWidgetMemberReorder`，异源 `CopyFilesToFolder` + `Refresh`

**LuaScript** 不继承 Container，只继承 Widget（Item）。不可放，不可拖入。

### 3.3 Slot（兼 drop target）

```cpp
enum class HitRegion { None, Empty, SortBefore, SortAfter, Handoff };

class Slot {
public:
    Slot(Container* parent, RECT bounds, size_t index);
    Container* GetParent() const;
    RECT GetBounds() const;
    Item* GetItem() const;
    void SetItem(Item*);       // null = 清空
    size_t GetIndex() const;
    bool IsEmpty() const;
    RECT GetIconRect() const;  // 64x64 居中 + top 6px

    // Drop target（取代独立 DropZone 类）
    HitRegion HitTest(POINT pt) const;
    std::wstring GetDropHint(HitRegion region, const std::vector<Item*>& sourceItems) const;
    void ExecuteDrop(HitRegion region, const std::vector<Item*>& sourceItems, Container* origin, int mods);
    void DrawDropIndicator(ID2D1DeviceContext* ctx, HitRegion region) const;
};
```

**HitTest 逻辑**（按优先级）：
1. `Handoff` — pt 在 icon rect + 4px padding 内，且 slot 非空 → "交给「xx」处理"
2. `SortBefore` — pt 在 slot 上半部分（或左半）→ "重新排序" / "插入到前"
3. `SortAfter` — pt 在 slot 下半部分（或右半）→ "插入到后"
4. `Empty` — slot 为空 → "移动到此空位"

**ListContainer 中 Container 与 Slot 交错排列**：
列表模式下，Container 的 chrome（标题栏、tab 栏）和 Slot 自然交错排布。插入位置由鼠标落在哪个 slot 的 `SortBefore`/`SortAfter` 决定，无需额外插入线对象。

**DrawDropIndicator**：
- GridContainer → VBar（竖杠，在 SortBefore=左 / SortAfter=右）
- ListContainer → HBar（横杠，在 SortBefore=上 / SortAfter=下）
- Empty → placeholder（蓝色虚线整格）
- Handoff → 不画

`GetIconRect()` 计算逻辑与现有 `GetItemIconRect(bounds)` 完全一致。

## 四、app.h 接入（最小化改动）

拖拽状态由 app 直接管理，不封装为独立类：

```cpp
class SnowDesktopApp {
    // 新成员
    std::vector<std::unique_ptr<Container>> containers_;  // DesktopGrid + 每个 WidgetContainer
    std::vector<std::unique_ptr<Item>> items_;            // 所有 DesktopIcon + FolderEntryIcon

    // 拖拽状态（取代 DragSource + DragOperation 两个类）
    std::vector<Item*> dragItems_;              // 正在被拖的 items
    Container* dragOrigin_ = nullptr;           // 来源容器
    Slot* dragTargetSlot_ = nullptr;            // 当前瞄准的 slot
    HitRegion dragTargetRegion_ = HitRegion::None;
    POINT dragStartPt_{}, dragCurrentPt_{};
    std::wstring dragHint_;

    void RebuildContainersAndItems();  // LayoutItems 末尾调用

    void OnMouseDown(POINT pt) {
        auto* item = FindItemAt(pt);
        if (item && item->IsSelected()) {
            dragItems_ = GetSelectedItems();
            dragOrigin_ = item->GetContainer();
            dragStartPt_ = pt;
        }
    }

    void OnMouseMove(POINT pt) {
        if (dragItems_.empty()) return;
        dragCurrentPt_ = pt;
        // 遍历 containers_ → BuildSlots() → slot.HitTest(pt)
        for (auto& c : containers_) {
            for (auto& slot : c->BuildSlots()) {
                auto region = slot->HitTest(pt);
                if (region != HitRegion::None) {
                    dragTargetSlot_ = slot.get();
                    dragTargetRegion_ = region;
                    dragHint_ = slot->GetDropHint(region, dragItems_);
                    return;
                }
            }
        }
        dragTargetSlot_ = nullptr;
        dragTargetRegion_ = HitRegion::None;
    }

    void OnLeftButtonUp(POINT pt) {
        if (dragTargetSlot_) {
            dragTargetSlot_->ExecuteDrop(dragTargetRegion_, dragItems_, dragOrigin_, GetModifiers());
        }
        dragItems_.clear();
        dragTargetSlot_ = nullptr;
    }

    void OnPaint(ID2D1DeviceContext* ctx) {
        for (auto& c : containers_) c->DrawContents(ctx);
        for (auto& item : items_) {
            if (item->IsSelected() && !dragItems_.empty()) continue;
            item->Draw(ctx, item->GetBounds(), item->IsSelected() ? 2 : 0);
        }
        // 拖拽中绘制
        if (!dragItems_.empty()) {
            POINT offset = { dragCurrentPt_.x - dragStartPt_.x, dragCurrentPt_.y - dragStartPt_.y };
            for (auto* item : dragItems_)
                item->Draw(ctx, OffsetRect(item->GetBounds(), offset), 3);
            if (dragTargetSlot_)
                dragTargetSlot_->DrawDropIndicator(ctx, dragTargetRegion_);
        }
    }
};
```

## 六、关键设计决策

### 6.1 Item 用包装器而非继承
`DesktopItem` 和 `FolderEntry` 保持原样（PIDL/Shell 耦合不改），用 `DesktopIcon`/`FolderEntryIcon` 包装。优势：不影响 JSON 序列化、Shell 操作、GDI 渲染。

### 6.2 Container 子类负责自己的文件操作
`FolderMapping::OnItemsDropped` 直接调用 `CopyFilesToFolder`、`FinishWidgetMemberReorder` 等——不需要外部 switch。这些函数作为 app.h 的 public helper 暴露给子类。

### 6.2.1 Container 布局分两类

**GridContainer（二维）**：只有 `DesktopGrid` 使用。`BuildSlots()` 按 `GridPage::columns × rows` 双重循环计算每个 cell 的 RECT。插入线默认 VBar。**可接受任意 Item 包括 Widget**——桌面既是图标的网格也是组件的网格。

**ListContainer（一维）**：所有 Widget 使用。`BuildSlots()` 按 `index × itemHeight` 线性计算。插入线默认 HBar。**不接受 Widget 放置**——Widget 之间不能嵌套。渲染形态运行时决定：
- `FixedList` — 内容不超出可见区，无滚动
- `ScrollingIconList` — 图标模式 + 滚动条
- `ScrollingRowList` — 列表行模式 + 滚动条

Widget 整体拖拽只能放到 DesktopGrid 上；文件/条目可以在任意 Container 间传递。

### 6.2.2 Widget 拆为 Widget / WidgetContainer

`LuaScript` 不需要 Container 能力。将原 Widget 拆为：
- `Widget : Item` — 纯 Item，可拖拽、可渲染
- `WidgetContainer : Widget, Container` — Item + Container，可拖可放

`Collection`、`FileCategories`、`FolderMapping` 继承 `WidgetContainer`。`LuaScript` 只继承 `Widget`，不声明 `BuildSlots` / `OnItemsDropped`。

### 6.3 拖拽源就是选中的 Item 数组

不需要独立的 DragSource 类。拖拽源由三个 app 变量描述：
- `dragItems_` (vector<Item*>) — 正在被拖的 items
- `dragOrigin_` (Container*) — 来源容器（null=桌面）
- `dragStartPt_` (POINT) — 拖拽起点

`Item::Draw(state=3, rect+offset)` 处理拖拽中的偏移渲染。`Item::CreateDataObject()` 处理 OLE 数据对象。

### 6.4 渲染 OO 化

每帧渲染路径：

```
OnPaint:
  for each Container c:
    c->DrawChrome(ctx, mousePt)       ← 滚动条、切换按钮、标题栏
    for each Slot s in c->GetSlots():
      if Item* item = s->GetItem():
        item->Draw(ctx, s->GetBounds(), state)  ← 图标+文本+选中高亮
  if !dragItems_.empty():
    for item in dragItems_: item->Draw(ctx, rect+offset, state=3)  ← 拖拽中 items 偏移绘制
    dragTargetSlot_->DrawDropIndicator(ctx, dragTargetRegion_)      ← 插入指示线
```

各对象的渲染方法：

| 对象 | 方法 | 渲染内容 | 实现来源 |
|---|---|---|---|
| `Item` | `Draw(ctx, rect, state)` | 图标+文本+选中高亮+hover效果 | 复用现有 `DrawD2DItemGeneric` / `DrawD2DItemGenericList` |
| `Container` | `DrawChrome(ctx, mousePt)` | 每个子类不同 | 见下表 |
| `Container` | `DrawContents(ctx)` | 遍历 Slot 调 Item::Draw | 基类实现 |
| `Slot` | `DrawDropIndicator(ctx, region)` | 插入线 | HitRegion + Container 类型决定 |

**DrawChrome 各子类：**

| Container 子类 | DrawChrome 内容 |
|---|---|
| `DesktopGrid` | 页面导航按钮 |
| `Collection` | 列表/图标切换按钮、标题栏、渐变底栏 |
| `FileCategories` | 分类 Tab 标签栏、列表/图标切换按钮 |
| `FolderMapping` | 列表/图标切换按钮、打开源文件夹按钮、标题栏、渐变底栏、滚动条 |
| `LuaScript` | 不实现（Lua 自渲染，Widget 不是 Container） |

**DrawDropIndicator 各 region：**

| HitRegion | GridContainer 渲染 | ListContainer 渲染 |
|---|---|---|
| `SortBefore` | VBar（竖杠在 item 左侧） | HBar（横杠在行上方） |
| `SortAfter` | VBar（竖杠在 item 右侧） | HBar（横杠在行下方） |
| `Empty` | Placeholder（蓝色虚线整格） | HBar（整行横杠） |
| `Handoff` | 不画 | 不画 |

**Item::Draw 的 state 参数**：
- `state=0 Normal` — 标准图标+文字
- `state=1 Hovered` — 加半透明 hover 背景
- `state=2 Selected` — 灰色半透明选中框（icon+text 区域）
- `state=3 Dragged` — 选中框+半透明（拖拽中跟鼠标的那个）

**现有渲染函数复用**：
- `DrawD2DItemGeneric` / `DrawD2DItemGenericList` → 移入 `Item::Draw`
- `DrawD2DScrollbar` → 移入 `Container::DrawChrome`（FolderMapping）
- `DrawD2DRoundedRectangle` / `DrawD2DFilledRectangle` → 保留为工具函数
- `GetItemIconRect` / `GetItemTextRect` → 移入 `Slot` 或 `Item`

### 6.5 Slot 每次 BuildSlots 重建
Slot 是轻量对象（RECT + 指针），每次 `BuildSlots()` 重建开销 < 1ms。不需要缓存/增量更新。

| 现有过程式代码 | 迁移到 |

## 八、拖拽行为矩阵（OO 设计视角）

> 每行标注该场景涉及的核心类 → 方法，展示多态分发路径。

### 8.8 M → 同M（映射文件夹内排序）

**不涉及文件操作**，纯内存重排。

### 8.15 组件整体拖拽（Widget as Item）

Widget 作为 Item 拖拽时，**只能放到 GridContainer（DesktopGrid）上**，不能放到 ListContainer（其他 Widget）上。

| # | 触发链 | 提示 | 执行 |
|---|---|---|---|
| W-1 | 点击 Widget 标题栏 → Widget 选中 → 拖到 DesktopGrid 空 cell | "移动组件" | `PlaceWidgetWithDisplacement` |
| W-2 | 拖到 DesktopGrid 已占 cell | "此处已被占用" | 不放置 |
| W-3 | 拖到 ListContainer（任意 Widget 内） | 无响应（不命中） | 不放置——Widget 不可嵌套 |
| W-4 | 拖边缘 | "调整组件大小" | ResizeWidget（沿用现有） |

### 8.16 修饰键行为

所有 `Container::OnItemsDropped()` 和 `Slot::ExecuteDrop(Handoff)` 接收 `int mods` 参数。文件操作时统一：

| mods | Disk operation |
|---|---|
| 0（默认） | `CopyFileEx` with MOVEFLAG if same drive |
| MK_CONTROL | `CopyFileEx` always copy |
| MK_ALT | `IShellLink` → `IPersistFile::Save` |

### 8.17 命中优先级

`OnMouseMove` 遍历 Slot HitRegion 的顺序：
1. 优先 `Handoff`（范围小，图标 rect + 4px）
2. 其次 `SortBefore` / `SortAfter` / `Empty`（范围大，整 slot）
3. 同一点优先返回 Handoff → 用户指向图标时触发 handoff，指向空白时触发排序

### 8.18 类方法对照速查

| 场景组合 | 涉及的类→方法 |
|---|---|
| 所有空 cell drop | `Slot::ExecuteDrop(Empty)` → `Container子类::OnItemsDropped` |
| 所有排序 drop | `Slot::ExecuteDrop(SortBefore/SortAfter)` → `Container子类::OnItemsDropped`（同源重排 or 异源插入） |
| 所有 handoff | `Slot::ExecuteDrop(Handoff)` → `ShellExecuteEx` 打开/处理 |
| 所有跨组件文件复制 | 目标 `Container::OnItemsDropped` 内调用 `CopyFilesToFolder` |
| 所有组件内重排 | 目标 `Container::OnItemsDropped` 内调用 `Reorder*` |
| 外部拖入 | `IDropTarget::Drop` → 解析 HDROP → `dragItems_` → `Slot::ExecuteDrop` → 同上分发 |
